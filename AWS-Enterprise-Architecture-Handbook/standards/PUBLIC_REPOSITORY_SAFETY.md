# Public Repository Safety

Before publication, remove or replace:

- AWS account IDs
- access keys, tokens, passwords, client secrets
- private and public IP addresses tied to real systems
- internal DNS names and URLs
- customer names and proprietary architectures
- CUI, PII, PHI, PCI data
- screenshots showing credentials, email addresses, usernames, or tickets
- production log events that expose identifiers

Use placeholders such as `<account-id>`, `<region>`, `<vpc-id>`, and synthetic company names.
