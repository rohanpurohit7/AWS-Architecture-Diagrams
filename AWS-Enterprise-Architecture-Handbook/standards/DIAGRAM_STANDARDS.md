# Diagram Standards

Every architecture diagram should show:

- trust boundaries
- account and VPC boundaries
- public and private subnets
- control plane versus data plane
- encryption points
- logging paths
- failure domains
- identity and authorization decisions
- ingress and egress paths
- human approval points for AI agents

Use consistent labels, left-to-right data flow, and a legend. Mermaid is useful for source-controlled flowcharts; SVG is preferred for polished portfolio visuals.
